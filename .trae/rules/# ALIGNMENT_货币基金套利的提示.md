# ALIGNMENT_货币基金套利的提示.md

## 1. 项目和任务特性规范

### 1.1 项目概述
- **项目名称**：货币基金套利提示系统
- **任务目标**：监控银华日利（511880）和华宝添益（511990）的买入赎回收益率，及时提醒套利机会

### 1.2 技术栈
- **前端**：Vue.js 3 + Vite + Element Plus（免费开源）
- **后端**：Python + FastAPI（免费开源）
- **数据存储**：SQLite（免费开源）
- **数据来源**：
  - 价格数据：AKSHARE库（免费开源）
  - 净值数据：天天基金网 + 网页爬虫（免费）
- **提醒方式**：企业微信机器人（免费）

### 1.3 架构模式
- **架构风格**：前后端分离
- **部署方式**：本地部署

## 2. 原始需求
- 查看银华日利和华宝添益的买入赎回收益率情况
- 每5分钟定时采集市场价格数据
- 基金净值数据当天采集一次
- 计算收益率并在界面展示
- 当出现套利机会时通过微信提醒
- 存储历史数据用于趋势分析

## 3. 边界确认

### 3.1 功能边界
- **做什么**：
  - 监控银华日利（511880）和华宝添益（511990）两支货币基金
  - 定时采集价格和净值数据
  - 计算买入赎回收益率（年化），包括买一价格和卖一价格的收益率计算
  - 界面展示实时数据和历史趋势，包括价格数据
  - 通过企业微信机器人发送提醒（年化收益率达到1.5%时）
  - 存储历史数据（5天）
  - 数据采集失败时进行报错

- **不做什么**：
  - 不支持其他货币基金
  - 不提供自动交易功能
  - 不支持云端部署
  - 不提供付费数据服务

### 3.2 技术边界
- 使用免费开源技术和工具
- 本地部署，不依赖外部付费服务
- 数据采集频率：价格5分钟/次，净值1天/次
- 提醒方式：仅企业微信机器人
- 提醒阈值：年化收益率达到1.5%
- 历史数据存储周期：5天

## 4. 需求理解

### 4.1 业务域理解
- **核心业务**：货币基金套利机会监控
- **关键指标**：买入赎回年化收益率（基于买一价格和卖一价格）
- **业务流程**：数据采集 → 计算年化收益率 → 展示 → 提醒（当年化收益率≥1.5%时）→ 异常处理（数据采集失败时报错）

### 4.2 数据模型理解
- **价格数据**：基金代码、时间戳、买一价格、卖一价格
- **净值数据**：基金代码、日期、净值
- **收益率数据**：基金代码、时间戳、买一价格年化收益率、卖一价格年化收益率
- **提醒记录**：时间戳、基金代码、年化收益率、提醒内容
- **错误记录**：时间戳、错误类型、错误信息

## 6. 年化收益率计算方式
年化收益率计算公式：[(净值 - 价格) / 价格] × (365/持有天数) × 100%
其中：
- 价格分别使用买一价格和卖一价格进行计算
- 持有天数默认为1天，即假设T日买入，T+1日赎回，计算得出的是年化收益率
